# ODL Color System

This document contains all color definitions for the ODL (Object Design Language) system across Light Mode, Dark Mode, and High Contrast modes.

## Color Token Structure

Each color token follows this structure:
- **Token Name**: Light Mode | Dark Mode | High Contrast

## Implementation Notes

### CSS Variables
```css
/* Example implementation */
:root {
  /* Brand Colors */
  --brand-build: #5DA10C;
  --brand-contentsolutions: #0B77D8;
  /* ... other colors */
}

/* Dark Mode */
[data-theme="dark"] {
  --info: #1B2E4A;
  --grey-50: #CCCDCE;
  /* ... other dark mode colors */
}

/* High Contrast Mode */
[data-theme="high-contrast"] {
  --grey-500: #000000;
  --grey-700: #000000;
  /* ... other high contrast colors */
}
```

### JavaScript/TypeScript
```typescript
export const colors = {
  light: {
    info: '#E0F3FF',
    grey50: '#FAFAFA',
    // ... other light colors
  },
  dark: {
    info: '#1B2E4A',
    grey50: '#CCCDCE',
    // ... other dark colors
  },
  highContrast: {
    info: '#E0F3FF',
    grey50: '#FAFAFA',
    // ... other high contrast colors
  }
};
```

### SCSS/Sass
```scss
// Brand colors (consistent across themes)
$brand-build: #5DA10C;
$brand-contentsolutions: #0B77D8;

// Theme-specific colors
@mixin light-theme {
  --info: #E0F3FF;
  --grey-50: #FAFAFA;
}

@mixin dark-theme {
  --info: #1B2E4A;
  --grey-50: #CCCDCE;
}
```

## Accessibility Guidelines

1. **High Contrast Mode**: Ensure all text meets WCAG AAA standards (7:1 for normal text, 4.5:1 for large text)
2. **Color Blindness**: Test all color combinations with color blindness simulators
3. **Focus Indicators**: Use appropriate contrast for focus states
4. **Semantic Colors**: Use status colors consistently (green for success, red for error, etc.)
